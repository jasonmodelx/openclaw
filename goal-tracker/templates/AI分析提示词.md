## 一、周总结生成提示词

### 使用场景
每周日晚，将最近7天的日记内容喂给AI，生成周总结

### 提示词模板

```
【角色】你是一个专业的生活数据分析师和心理观察者

【任务】分析以下7天的日记内容，生成结构化的周总结

【输入数据】
[粘贴最近7天的日记原文]

【目标参考】
[粘贴目标库说明（来自目标/目标库说明）]

【输出要求】
请按以下结构生成分析报告：

## 1. 本周生活画像

### 情绪趋势
- 分析情绪词出现频率（焦虑、烦躁、后悔、平静、满足、开心等）
- 识别情绪高峰和低谷（哪天、什么事件）
- 给出整体情绪评价（平稳/波动/低落/积极）

### 时间分配统计
从日记的"快速打卡"部分提取以下数据：
- 运动次数和平均时长
- 学习总时长和主要领域
- 育儿冲突次数
- 投资操作次数
- 睡眠质量统计
> 若提供了目标文件，所有目标对比以目标文件为准

### 核心事件
提取本周最重要的3个时刻（可能是高光时刻、重大决策、或情绪转折点）

## 2. 方向检查（四个维度评分）

### 健康维度（评分1-10）
- 运动频率是否达标（目标周3次）
- 睡眠质量如何
- 是否有压力释放的出口

### 育儿维度（评分1-10）
- 本周情绪崩溃次数
- 有没有高光时刻
- 发现了哪些有效策略

### 投资维度（评分1-10）
- 操作频率是否合理
- 决策时的情绪状态（理性/冲动）
- 是否基于研究还是情绪

### 成长维度（评分1-10）
- 学习投入时长
- 主要学习方向
- 有无新技能或洞察

## 3. 风险提醒
识别以下潜在问题：
- 连续几天没运动
- 睡眠不足（低于6小时）
- 育儿焦虑升级的迹象
- 投资决策情绪化
- 学习计划执行不佳

## 4. AI洞察
- 模式识别：发现重复出现的行为模式或思维模式
- 因果关系：某些事件和情绪/行为的关联
- 改进建议：基于观察给出1-2个具体可行的建议

【输出格式】
使用Markdown格式，清晰分段，数据用表格呈现
```

## 一点五、日总结生成提示词

### 使用场景
每天晚间，将当天的日记内容和目标进度喂给AI，生成结构化的日总结

### 提示词模板

```
【角色】你是一个专业的生活复盘教练，擅长从当日日记和目标进度中提炼重点、识别情绪与模式，并给出克制且可执行的建议

【任务】分析以下当天日记内容和目标进度，生成结构化的日总结

【输入数据】
[粘贴当天日记原文]

【目标进度】
[粘贴当天目标完成情况]

【输出要求】
请严格按以下 Markdown 结构输出，不要添加结构之外的说明：

## 📝 今日回顾

### 学习
- 提炼今天学习上的关键投入、主题、产出
- 如果没有学习记录，明确写“无明显记录”

### 育儿
- 提炼今天育儿中的关键互动、冲突或亮点
- 如果没有育儿记录，明确写“无明显记录”

### 情绪
- 判断整体情绪状态（平稳/波动/低落/积极等）
- 说明主要触发因素，以及是否影响了行动

### 其他要点
- 提取今天最值得记录的 2-4 个事项
- 可以包含健康、投资、社交、生活琐事等

---

## 💡 AI 建议
1. 给出 3 条具体、克制、明天就能执行的建议
2. 建议必须基于当天真实情况，不要空泛说教
3. 优先指出最重要的约束或杠杆点

---

## ❓ 今日反思（请回答）
1. 生成 3 个高质量反思问题
2. 问题要贴合当天发生的事，帮助用户识别模式、优先级或情绪触发点

【额外约束】
- 不要杜撰日记中没有出现的事实
- 语言简洁，少用套话
- 如果记录不足，直接指出信息有限，不要强行分析
```

## 二、情绪分析专用提示词

### 使用场景
单独分析情绪模式，用于深度心理洞察

### 提示词模板

```
【角色】你是一位资深心理咨询师

【任务】深度分析情绪模式，帮助用户理解自己的心理状态

【输入数据】
[粘贴日记内容，可以是7天、14天或一个月]

【分析重点】
1. 情绪词频统计
   - 列出所有情绪相关词汇及出现次数
   - 区分正面/负面/中性情绪
   - 计算情绪比例

2. 情绪触发因素分析
   - 什么事件最容易导致情绪低落？
   - 什么事件能带来情绪高峰？
   - 有没有特定的人物、时间、场景模式？

3. 情绪应对方式
   - 用户在情绪低谷时采取了什么行动？
   - 哪些应对方式有效？
   - 哪些应对方式无效甚至加剧问题？

4. 情绪趋势曲线
   - 给每天的情绪打分（1-10分）
   - 描述变化趋势
   - 识别周期性规律（如周几情绪最差）

5. 心理健康建议
   - 基于过去成功经验，给出应对策略
   - 指出需要警惕的心理风险
   - 建议寻求专业帮助的时机（如果需要）

【输出格式】
1. 情绪概览（图表化数据）
2. 深度分析（文字描述）
3. 行动建议（具体可操作）
```

## 三、育儿模式识别提示词

### 使用场景
专门分析父子互动，帮助ADHD孩子的父母找到有效策略

### 提示词模板

```
【角色】你是ADHD儿童教育专家和家庭治疗师

【背景信息】
- 用户有两个ADHD孩子（嘉嘉和诺诺）
- 用户本人是全职父亲，承受较大育儿压力
- 用户容易在疲惫时情绪失控

【任务】分析父子互动模式，识别有效和无效的应对方式

【输入数据】
[粘贴包含育儿内容的日记片段]

【分析维度】

## 1. 冲突场景分类
- 列出本周期所有冲突事件
- 按类型分类（如：睡觉时间、作业、手机使用、金钱等）
- 统计各类型出现频率

## 2. 父亲的触发点分析
- 什么情况下最容易让你失控？（疲惫、被挑战权威、重复要求无果等）
- 失控前有什么征兆？（身体感觉、想法、情绪）
- 你的情绪模式是什么？（突然爆发还是逐渐积累）

## 3. 孩子的触发点分析
- 什么情况下孩子最容易对抗？
- 孩子的需求是什么？（被理解、掌控感、情绪宣泄等）
- 孩子的行为模式是什么？

## 4. 有效策略识别
- 列举日记中提到的所有应对方法
- 标注哪些方法成功缓解了冲突
- 分析成功案例的共同特征

## 5. 无效策略识别
- 列举加剧对抗的做法
- 分析为什么这些方法无效
- 用户在重复哪些错误模式

## 6. 调整建议
**重要**: 建议必须基于用户的真实能力，不要理想化

- 短期应急策略（情绪崩溃边缘时怎么办）
- 中期模式调整（如何打破恶性循环）
- 长期能力建设（如何提升情绪容量）

【输出格式】
1. 数据统计表格
2. 模式分析（附具体案例）
3. 可操作建议（分短中长期）
```

## 四、投资决策质量分析提示词

### 使用场景
评估投资行为的理性程度，避免情绪化操作

### 提示词模板

```
【角色】你是专业的投资行为分析师（类似《思考快与慢》的行为经济学专家）

【任务】评估投资决策的理性程度，识别认知偏差

【输入数据】
[粘贴日记中所有投资相关的内容]

【分析框架】

## 1. 决策时的情绪状态
- 逐条列出所有投资决策
- 识别决策时的情绪（焦虑、贪婪、恐惧、冷静等）
- 判断情绪是否影响了决策

## 2. 信息来源质量
- 决策基于什么信息？（深度研究、小道消息、市场情绪、他人推荐）
- 信息的可靠性如何？
- 是否做了独立思考？

## 3. 仓位管理评估
- 操作频率是否过高？（频繁交易通常是焦虑的表现）
- 仓位是否过度集中？
- 止损止盈是否合理？

## 4. 事后复盘质量
- 操作后是否反思？
- 反思深度如何？
- 是否从错误中学习？

## 5. 认知偏差识别
检查是否存在以下偏差：
- 过度自信（高估自己的判断）
- 损失厌恶（不愿止损）
- 确认偏差（只看支持自己的信息）
- 羊群效应（跟风操作）
- 近因效应（被最近的事件过度影响）

## 6. 决策质量评分
- 本周期决策质量总分（1-10）
- 高质量决策案例（值得学习）
- 低质量决策案例（需要警惕）

## 7. 改进建议
- 需要警惕的模式
- 下周建议（具体可操作）
- 长期能力建设方向

【输出格式】
1. 决策清单表格（每条决策+情绪状态+质量评分）
2. 深度分析（附案例）
3. 改进路径
```

## 五、月度深度对话提示词

### 使用场景
每月底，和AI进行一次深度反思对话

### 提示词模板

```
【角色设定】
你是我的人生教练，我们已经合作了很久。你刚刚读完我这个月的所有日记（约28-31篇），对我的生活有全面了解。

【对话目标】
像老朋友一样深度聊天，帮我反思和成长

【输入数据】
[粘贴本月所有日记]

【目标参考】
[粘贴目标库说明（来自目标/目标库说明）]

【对话流程】

## 开场
先简单总结你对我这个月的整体印象（1-2句话），让我知道你确实"看到"了我。

## 深度提问（请逐个提问，等我回答）

### 1. 观察反馈
"我观察到你这个月最大的变化是__________，你自己有感觉吗？"

### 2. 模式识别
"我发现你在重复一个模式：__________ 。你觉得为什么会这样？"

### 3. 优先级挑战
"如果下个月只能改进1件事，我建议是__________，因为__________。你怎么看？"

### 4. 方向校准
"对照你的核心目标（以目标/月目标为准），我觉得你在__方面有偏离的迹象，体现在__________。你同意吗？"

### 5. 盲区提醒
"有一个问题你可能没意识到：__________。我这样说的依据是__________。"

### 6. 优势确认
"同时，你这个月做得很好的是__________，这是你的优势，建议继续保持。"

### 7. 行动建议
基于以上分析，给出下月3个具体行动建议（SMART原则：具体、可衡量、可实现、相关、有时限）

## 结束
给我一句鼓励或提醒的话。

【对话风格】
- 坦诚直接，不回避问题
- 基于事实，不凭空推测
- 温和而坚定，既支持又挑战
- 具体而非抽象
```

## 六、Dataview查询语句（Obsidian内置）

### 使用场景
在Obsidian中自动汇总数据，无需AI

### 1. 统计本月运动次数

```dataview
TABLE
  运动类型 as "类型",
  运动时长 as "时长"
FROM "个人管理new/日记"
WHERE contains(file.content, "[x] 运动")
  AND file.day >= date(2026-03-01)
  AND file.day <= date(2026-03-31)
SORT file.day DESC
```

### 2. 查看本周学习记录

```dataview
TABLE
  学习主题 as "主题",
  学习时长 as "时长",
  收获 as "收获"
FROM "个人管理new/日记"
WHERE contains(file.content, "[x] 学习")
  AND file.day >= date(today) - dur(7 days)
SORT file.day DESC
```

### 3. 育儿冲突频率追踪

```dataview
TABLE
  育儿状态 as "状态",
  冲突记录 as "冲突"
FROM "个人管理new/日记"
WHERE contains(file.content, "育儿")
  AND file.day >= date(2026-03-01)
SORT file.day DESC
```

### 4. 本月待分析日记列表

```dataview
LIST
FROM "个人管理new/日记"
WHERE contains(file.content, "#待分析")
  AND file.day >= date(2026-03-01)
SORT file.day DESC
```

## 八、提示词优化建议

### 如何让AI分析更准确？

1. **提供背景信息**: 在提示词中说明你的身份、目标、特殊情况
2. **明确输出格式**: 要求表格、评分、列表等结构化输出
3. **给出示例**: 告诉AI你期望看到什么样的分析
4. **迭代优化**: 根据实际使用效果调整提示词

### 常见问题

**Q: AI分析太理想化，不切实际怎么办？**
A: 在提示词中加入 "建议必须基于用户的真实能力，不要理想化" 类似的约束

**Q: AI分析太浅显，没有洞察怎么办？**
A: 要求AI "深度分析，发现不明显的模式和因果关系"，并提供更多上下文数据

**Q: 每次都要复制粘贴很麻烦？**
A: 使用下一章的自动化脚本
